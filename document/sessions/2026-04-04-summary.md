### H-07: crash-recovery.ts ロジック修正
対象ファイル: `packages/api/src/engine/crash-recovery.ts`
問題: 行66-70 で `recovering` に設定後、行86-89 で即 `idle` に上書き → recovering 状態が意味をなさない
修正方針: `recovering` 設定後、heartbeat-engine.ts の `runAgentHeartbeat()` を呼び出し、
その結果に応じて `idle`（成功）or `crashed`（失敗）に更新する

### H-08: budget-monitor.ts SQL 修正
対象ファイル: `packages/api/src/engine/budget-monitor.ts`
問題: `leftJoin(agents, ...)` → エージェント削除済みの場合 `agents.company_id` が NULL になり、
コスト計算から漏れる（削除済みエージェントのコストが集計されない）
修正方針: `innerJoin` に変更してエージェントが存在する分のみカウント、
または `cost_events` に `company_id` を直接持たせる

### H-09: パスワード強度チェック強化
対象ファイル: `packages/api/src/middleware/validate.ts`
問題: `isStrongPassword` が `length >= 8 && length <= 128` のみ（`12345678` が通過）
修正方針: 大文字・小文字・数字・記号のうち3カテゴリ以上を要求

```typescript
// 現状（問題あり）
export function isStrongPassword(password: string): boolean {
  return password.length >= 8 && password.length <= 128;
}

// 修正後イメージ
export function isStrongPassword(password: string): boolean {
  if (password.length < 8 || password.length > 128) return false;
  const categories = [/[A-Z]/, /[a-z]/, /[0-9]/, /[^A-Za-z0-9]/];
  const matched = categories.filter(r => r.test(password)).length;
  return matched >= 3;
}
```

### H-10: routines.ts PATCH エンドポイント追加
対象ファイル: `packages/api/src/routes/routines.ts`
問題: PATCH /:routineId が存在しない → name / description / cron_expression / enabled の更新が不可能
修正方針: PATCH を追加。sanitizeString + isValidCronExpression + ownership チェックを含む

### H-11: CLI agent.ts type バグ修正
対象ファイル: `packages/cli/src/commands/agent.ts`
問題: `let type: AgentType | undefined` で初期化 → `if (!name || !type)` の `!type` が常に true
→ `--type` を渡しても inquirer が常に起動する
修正方針:
```typescript
// 修正前（バグあり）
let type: AgentType | undefined;
if (!name || !type) { ... }

// 修正後
let type = options.type as AgentType | undefined;
if (!name || !type) { ... } else { /* type は既に設定済み */ }
```
